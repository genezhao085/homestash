# HomeStash 代码审查标准

## 审查流程
1. 开发者完成功能 → 运行 `requesting-code-review`
2. 审查 Agent 独立审查变更
3. 审查通过 → 创建 PR → 等待 CI
4. CI 通过 → 合并到 main

## 审查检查清单

### 功能性
- [ ] 功能是否按 PRD 实现？
- [ ] 边界条件是否处理？（空值、极限值、网络异常）
- [ ] UI 状态是否完整？（加载中、空数据、错误）

### 安全性
- [ ] 无硬编码密钥或 Token
- [ ] 无 `eval()` / `exec()` 调用
- [ ] 无 SQL 注入风险（使用参数化查询）
- [ ] 文件路径不硬编码

### 可维护性
- [ ] 命名清晰：变量名表达意图
- [ ] 函数单一职责：不超过 50 行
- [ ] 重复代码已提取为公共方法
- [ ] 复杂逻辑有注释说明

### 测试
- [ ] 核心逻辑有单元测试
- [ ] 测试覆盖正常路径 + 异常路径
- [ ] 测试可重复运行（无外部依赖假死）

### Flutter 特有问题
- [ ] 无 `BuildContext` 跨异步使用
- [ ] Controller 在 `dispose()` 中释放
- [ ] `setState()` 前检查 `mounted`
- [ ] 列表项有唯一 key
